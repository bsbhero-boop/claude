# 파일명 고치기 — 맥용

맥에서 만든 파일을 윈도우·리눅스로 옮기거나 웹에 올리면 한글 파일명이
`ㅍㅏㅇㅣㄹ.txt` 처럼 깨져 보이는 문제를 고치는 **오프라인 드롭 앱**입니다.

> 파일이 이미 **윈도우에 있고 윈도우에서 고칠 거라면** 이 폴더가 아니라
> [`../windows/`](../windows/) 를 사용하세요. 이 폴더는 **맥에서** 실행하는 버전입니다.

## 왜 깨지나요?

macOS는 한글 파일명을 유니코드 **NFD(자모 분리형)** 으로 저장합니다.
`한` 을 `ㅎ+ㅏ+ㄴ` 처럼 낱자로 쪼개서 보관하죠. 반면 윈도우·리눅스·대부분의
웹 서비스·압축 프로그램은 **NFC(완성형)** 을 기대하기 때문에, 맥에서 만든
이름이 다른 환경에서 깨져 보입니다.

이 앱은 파일·폴더 이름을 **NFC(완성형)** 으로 바꿔서 이 문제를 해결합니다.

## 특징

- **드래그 앤 드롭** 한 번이면 끝 — 다른 기능 없음
- 사본을 만들지 않고 **같은 자리에서 이름만** 바꿈 (원하시면 아래 CLI로 사본 방식도 가능)
- **폴더는 통째로** 드롭 가능 (안의 파일·하위 폴더까지 재귀적으로 처리)
- **완전 오프라인** — 인터넷 불필요, 맥 기본 도구(perl/python3)만 사용
- 이미 정상(NFC)인 파일은 건드리지 않고, 같은 이름이 이미 있으면 덮어쓰지 않고 건너뜀

## 설치(앱 만들기)

Xcode 없이 맥 기본 도구만으로 앱을 만듭니다. 터미널에서:

```bash
cd filename-fixer/mac
./build.sh
```

같은 폴더에 **`파일명 고치기.app`** 이 생깁니다. 응용 프로그램 폴더나 Dock으로
옮겨 두고 쓰세요.

> 처음 실행할 때 데스크탑/문서/다운로드 폴더 접근 권한을 물으면 **[허용]** 을
> 누르세요. (macOS 개인정보 보호 기능이며, 이 앱은 인터넷으로 아무것도 보내지 않습니다.)

## 사용법

깨진 파일명을 가진 **파일이나 폴더를 앱 아이콘 위로 드래그**해서 놓으면,
같은 자리에서 파일명이 완성형(NFC)으로 바뀌고 결과 요약이 뜹니다.

## 터미널에서 바로 쓰기 (앱 없이)

빌드가 귀찮으면 엔진 스크립트를 바로 실행해도 됩니다.

```bash
# 파일 여러 개
./fixnames.sh ~/Downloads/깨진파일.txt ~/Desktop/사진.jpg

# 폴더 통째로 (안의 모든 파일·폴더 재귀 처리)
./fixnames.sh ~/Downloads/내폴더
```

## 파일 구성

```
mac/
├─ fixnames.sh            # 실제 이름 변경 엔진 (NFC 정규화 + 안전한 이름 변경)
├─ FixFilename.applescript # 드롭 앱 껍데기 (드롭된 항목을 엔진에 전달)
├─ build.sh              # osacompile 로 "파일명 고치기.app" 생성
└─ README.md
```

## 자주 묻는 질문

**Q. 파일 내용이 바뀌나요?**
아니요. **이름만** 바꿉니다. 내용·수정일·권한은 그대로입니다.

**Q. 안전한가요? (되돌릴 수 있나요?)**
같은 이름이 이미 있으면 덮어쓰지 않고 건너뛰며, 이름 변경은 임시 이름을 거쳐
두 단계로 안전하게 처리합니다. 눈에 보이는 글자는 그대로이고 내부 표현(NFC)만
바뀌므로 사실상 원래 이름과 동일합니다.

**Q. 어떤 저장소(디스크)에서 되나요?**
요즘 맥의 기본 디스크(APFS)와 윈도우 공유용 USB(exFAT)에서 잘 동작하며,
바뀐 이름이 그대로 유지됩니다. (아주 오래된 HFS+ 포맷 디스크는 OS가 이름을
자동으로 다시 분해하므로, 그 디스크 자체에서는 효과가 제한적일 수 있습니다.
보통 다른 환경으로 옮길 때 문제가 되므로 이 경우 압축·전송 시점에 처리하세요.)

**Q. "확인되지 않은 개발자" 경고가 떠요.**
직접 빌드한 앱이라 서명은 없지만, 로컬에서 만들었기 때문에 보통 경고 없이
실행됩니다. 혹시 경고가 뜨면 앱을 **우클릭 → 열기** 로 한 번만 허용하세요.
