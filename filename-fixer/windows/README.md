# 파일명 고치기 — 윈도우용

맥에서 만든 파일을 윈도우로 옮겼더니 한글 파일명이 `ㅍㅏㅇㅣㄹ.txt` 처럼
깨져 보이는 문제를 고칩니다. **드래그 앤 드롭** 한 번이면 끝입니다.

## 왜 깨지나요?

macOS는 한글 파일명을 유니코드 **NFD(자모 분리형)** — `한` 을 `ㅎ+ㅏ+ㄴ` 으로
쪼갠 형태 — 로 저장합니다. 이 파일을 윈도우로 옮기면 윈도우는 **NFC(완성형)** 를
기대하기 때문에 이름이 깨져 보이거나 정렬이 이상해집니다. 이 도구는 파일·폴더
이름을 **NFC** 로 바꿔서 문제를 해결합니다.

## 준비물

- 윈도우 10 / 11 (별도 설치 **불필요** — 윈도우에 기본 포함된 PowerShell 사용)
- 인터넷 연결 필요 없음 (완전 오프라인)

## 사용법

1. 이 폴더의 **`FixNames.bat`** 과 **`FixNames.ps1`** 두 파일을 같은 폴더에 둡니다.
   (예: 바탕화면에 함께 복사)
2. 깨진 파일명을 가진 **파일이나 폴더를 `FixNames.bat` 아이콘 위로 드래그**해서 놓습니다.
3. 검은 창이 뜨면서 이름이 완성형(NFC)으로 바뀌고, 처리 결과가 표시됩니다.
   아무 키나 누르면 창이 닫힙니다.

> **폴더를 통째로** 올려도 됩니다. 안의 모든 파일과 하위 폴더 이름까지 재귀적으로 고칩니다.

### 더 편하게 — "보내기" 메뉴에 등록 (선택)

매번 파일을 끌어다 놓기 번거로우면, 우클릭 **보내기** 메뉴에 넣어 두세요.

1. `Win + R` → `shell:sendto` 입력 → 엔터 (SendTo 폴더가 열립니다)
2. `FixNames.bat` 의 **바로 가기**를 만들어 이 폴더에 넣습니다.
3. 이제 아무 파일/폴더나 **우클릭 → 보내기 → FixNames** 로 바로 처리할 수 있습니다.

## 안전한가요?

- **이름만** 바꿉니다. 파일 내용·수정일은 그대로입니다.
- 같은 이름이 **다른 파일로 이미 있으면 덮어쓰지 않고 건너뜁니다**(데이터 손실 없음).
- 이미 정상(NFC)인 파일은 건드리지 않습니다.

## 명령줄에서 직접 실행 (앱 없이)

```powershell
powershell -ExecutionPolicy Bypass -File FixNames.ps1 "C:\경로\깨진폴더"
powershell -ExecutionPolicy Bypass -File FixNames.ps1 "C:\a\파일1.txt" "C:\b\파일2.jpg"
```

## 문제 해결

- **"이 앱은 보호를 위해 차단되었습니다" / SmartScreen 경고**
  `.bat` 을 우클릭 → 속성 → 아래쪽 **[차단 해제]** 체크 → 확인 후 다시 실행하세요.
- **PowerShell 실행 정책 오류**
  `.bat` 은 `-ExecutionPolicy Bypass` 로 실행하므로 보통 문제가 없습니다.
  직접 `.ps1` 을 실행할 때만 위 명령줄 예시처럼 `-ExecutionPolicy Bypass` 를 붙이세요.
- **창의 한글이 깨져 보임**
  `.bat` 이 `chcp 65001`(UTF-8)로 콘솔을 설정합니다. 그래도 깨지면 콘솔 글꼴을
  "굴림체"나 한글 지원 글꼴로 바꿔 보세요. (이름 변경 자체는 정상 동작합니다.)

## 파일 구성

```
windows/
├─ FixNames.bat   # 드래그 대상 (드롭된 경로를 PowerShell 에 전달)
├─ FixNames.ps1   # 실제 이름 변경 엔진 (NFC 정규화 + 안전한 이름 변경)
└─ README.md
```
