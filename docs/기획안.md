## 1. 사업 개요

### 1.1 배경
- 혼자 병원 이용이 어려운 어르신(특히 퇴원 직후 회복기 환자)의 안전한 의료 이용을 위해 **병원동행인력(돌봄제공인력)**이 전 과정을 동행 지원한다.
- 병원동행인력은 **의료인이 아니며**, 이동·절차·정서·안전을 지원하되 **의료행위·대리결정·금전대납·개인정보 유출** 등은 절대 수행할 수 없다.
- 기존 집합·이론 교육만으로는 실제 현장의 *순간 판단*(낙상 위험, 응급 증세, 개인정보 노출, 자기결정 존중 등)을 체득하기 어렵다.

### 1.2 목적
> "설명을 듣는 교육"에서 → **"직접 해보며 몸에 익히는 체험"**으로.

가상 병원에서 **'하루 동행'** 전 과정을 직접 수행하며, 실수해도 부담 없이 반복 연습하고, 잘못된 선택에는 즉시 이유와 수칙을 피드백받아 **현장에서 바로 통하는 판단력**을 기른다.

### 1.3 주요 학습자 특성과 설계 반영
| 특성 | 설계 반영 |
|---|---|
| **50대 이상 다수** | 큰 글자·큰 버튼, 한 화면 1과제, 단순 메뉴 |
| 스마트폰 게임 조작에 익숙하지 않음 | 조작은 **탭(터치)만** — 복잡한 제스처·조준·타이밍 없음 |
| 노안·저시력 가능성 | 최소 18px 본문, 20~24px 버튼, 고대비 색, **읽어주기(TTS)** |
| 학습 시간 분절(자투리 시간) | 모듈 단위(3~7분), **중간 저장**, 이어하기 |
| 실패에 대한 부담 | **연습 모드**(점수 부담 없음) 우선, 틀려도 "다시 생각해 볼까요?" 톤 |

### 1.4 지원 환경
- **모바일 세로 화면 우선**(반응형), 태블릿·PC 겸용
- 웹 기반(설치 불필요), 최신 모바일 브라우저
- **3단계 폴백**: 고품질 3D → 저사양 간소화(박스형) 3D → WebGL 미지원 시 2D 텍스트/버튼 모드 (모든 학습 내용 100% 이용 가능)

## 2. 학습 목표 (Learning Objectives)

교육 매뉴얼 4개 장(Ⅰ~Ⅳ)을 학습 성취 목표로 재정의한다.

| 영역 | 학습 목표 | 매뉴얼 근거 |
|---|---|---|
| **역할 이해** | 병원동행 서비스의 목적과 돌봄제공인력의 4대 역할(이동·절차·정서·안전 지원)을 설명할 수 있다 | Ⅰ.1 |
| **절차 수행** | 병원 이용 7단계(접수→대기→진료→검사→수납→약국→귀가) 각 단계의 수행 업무를 순서대로 실행할 수 있다 | Ⅱ.2 |
| **키오스크 보조** | 접수·수납·증명서 발급 키오스크를 **어르신이 직접 하도록 보조**하며 개인정보 노출을 차단할 수 있다 | Ⅱ.2 |
| **안전관리** | 보행보조·휠체어·엘리베이터·문턱·경사로·승하차에서 낙상을 예방할 수 있다 | Ⅳ.1~2 |
| **역할 범위(금지행위)** | 의료행위·대리결정·대리서명·금전대납·개인정보유출 등 **수행 불가능 업무 6종**을 구분하고 거절할 수 있다 | Ⅰ.3, Ⅲ.4 |
| **응급대응** | 낙상·의식저하·호흡곤란·저혈당·경련 시 **초기 대응 5원칙**과 장소별 신고체계(병원 내 의료진 호출 / 병원 밖 119)를 적용할 수 있다 | Ⅳ.4~5 |
| **의사소통·정서** | 존중·차분·배려 태도로 이용자·보호자·의료진과 역할 범위를 지켜 소통할 수 있다 | Ⅲ.1~3 |

**성취 판정**: 각 목표는 해당 모듈의 체크리스트 수행 + 상황판단 정답률로 측정하며, §9·§12의 이수 기준에 연동한다.

## 3. 설계 원칙

1. **절차 우선(Procedure-first)**: 모든 모듈은 실제 동행 동선을 그대로 따른다. 학습 순서 = 현장 순서.
2. **능동 학습(Active)**: 읽고 넘기는 대신 *체크리스트를 직접 확인*해야, *선택을 직접 해야* 다음으로 진행(게이팅).
3. **즉시 피드백(Feedback)**: 올바른 선택 → 짧은 확인. 잘못된 선택 → **왜 안 되는지 + 올바른 대응 + 관련 수칙**을 즉시 제시.
4. **안전한 실패(Safe-to-fail)**: 연습 모드에서는 감점·탈락이 없다. 응급·금지행위 오답은 "위험 행동" 배지로 강하게 각인시키되 비난하지 않는다.
5. **저부하 3D(3D-lite)**: 공간 이해가 꼭 필요한 장면만 3D. 대화·판단은 카드형 UI. 모바일 60fps·저용량 우선.
6. **콘텐츠-코드 분리**: 모든 교육 문구·문항·체크리스트는 `js/data.js` 한 파일에서 관리 → 지자체·수행기관 지침 변경 시 개발자 없이 수정 가능.

## 4. 학습 시나리오 구조도

전체 프로그램은 **'가상 병원동행 하루 체험'** 한 편이며, 8개 모듈이 시간 순으로 이어진다.

```
[시작화면: 기관 로고 · 교육 시작]
        │
        ▼
[학습 모드 선택]  ── 연습 모드(부담 없음) / 평가 모드(이수용 채점)
        │
        ▼
┌───────────────────  가상 병원동행 하루 체험  ───────────────────┐
│                                                                │
│  ① 사전 준비 ──▶ ② 도착·접수 ──▶ ③ 진료대기 ──▶ ④ 진료 동행    │
│      (집/정문)      (로비·키오스크)   (대기실)       (진료실)        │
│                                                    │           │
│                                                    ▼           │
│  ⑦ 귀가·기록 ◀── ⑥ 수납·약국 ◀── ⑤ 검사 이동 ────┘           │
│    (차량/기록지)   (수납·약국)      (검사실)  ※검사·처방 없으면 분기 │
│      │                                                         │
│      ▼                                                         │
│  ⑧ 응급상황 대응 시뮬레이션 (낙상·의식저하/호흡곤란·저혈당·경련) │
│                                                                │
└────────────────────────────────────────────────────────────────┘
        │
        ▼
[수료 평가 화면]
  · 모듈별 점수 요약    · 취약 항목(틀린 수칙/위험행동)
  · 재학습 권장 모듈    · 이수 여부 · 수료증(선택)
```

**모듈 내부 공통 흐름(beat 구조)** — 각 모듈은 아래 유닛(beat)의 조합으로 구성된다.

```
intro(학습 목표 안내)
  → scene(3D 공간 진입·동선 이해)
  → check(체크리스트 직접 수행) ──반복──┐
  → decision("무엇을 해야 할까요?" 선택 + 즉시 피드백) ──반복──┤
  → [kiosk / dialogue / record 등 특수 유닛]                  │
  → 모듈 완료(점수·완료 저장) ◀───────────────────────────────┘
```

- **분기(branch)**: 모듈 5(검사)·모듈 6(약국)은 "검사 없음/처방 없음" 선택 시 건너뛴다 → 현장 변동성 반영.
- **재진입**: 어느 모듈이든 메뉴에서 다시 선택해 반복 학습 가능. 완료 모듈에는 ✓ 표시.

## 5. 모듈별 화면 흐름도

> 표기: `[화면/패널]` · `→` 진행 · `⤾` 오답 시 즉시 피드백 후 재선택 · `✅` 체크리스트 게이팅

### 모듈 1 · 사전 준비  🎒  (장소: 집·병원 정문)
```
[intro] 오늘의 목표: 정보·일정·준비물·비상연락망 확인
 → [scene] 정문에서 어르신을 맞이 (3D)
 → ✅[check] 준비물 점검: 신분증 / 진료카드·진료의뢰서 / 복용약 목록 / 진료비·교통비 / 보호자 연락처
 → [decision] "진료의뢰서를 집에 두고 오셨습니다. 어떻게 할까요?"  ⤾
 → [decision] "어르신이 '어제부터 어지럽다'고 하십니다."  ⤾
 → 모듈 완료
```

### 모듈 2 · 병원 도착 및 접수  🖥️  (장소: 로비·키오스크·원무과)
```
[intro] 접수창구 찾기·키오스크 보조·본인확인·대기번호·휠체어
 → [scene] 로비 → 키오스크 이동 (3D)
 → [kiosk] 진료 접수: 본인확인 → 진료과 선택 → 접수 확인 → 대기번호표
 → ✅[check] 접수 확인: 본인확인 완료 / 진료과·의료진 확인 / 대기번호표 수령 / 휠체어 필요 여부 판단
 → [decision] "뒷사람이 키오스크 화면을 볼 수 있습니다(개인정보)."  ⤾
 → [decision] "어르신이 '복잡하니 네가 다 눌러줘' 하십니다(자기결정)."  ⤾
 → 모듈 완료
```

### 모듈 3 · 진료대기  🪑  (장소: 대기실·화장실)
```
[intro] 대기 안내·호출 확인·화장실/휴게·상태 관찰·낙상 위험
 → [scene] 진료과 대기실 (3D)
 → ✅[check] 대기 중 업무: 대기번호·호출 확인 / 질문 미리 메모 / 상태 수시 관찰 / 낙상 위험 요소 확인
 → [decision] "어르신이 화장실에 가고 싶어 하십니다. 곧 순서일 수 있습니다."  ⤾
 → [decision] "대기 1시간째, 어르신이 피곤해 보이십니다."  ⤾
 → 모듈 완료
```

### 모듈 4 · 진료 동행  🩺  (장소: 진료실)
```
[intro] 진료실 이동·의사표현 보조·메모·다음 절차 / 의료판단 금지
 → [scene] 진료실 입장 (동의 확인) (3D)
 → ✅[check] 진료실 업무: 의사 설명 메모 / 준비한 질문 확인 / 다음 예약·검사 일정 기록 / 생활 주의사항 확인
 → [decision] "의사가 '어디가 불편하세요?' 물으십니다. 누가 답할까요?(증상 설명 주체)"  ⤾
 → [decision] "의사가 어려운 용어로 빠르게 설명합니다. 어르신이 못 알아들으십니다."  ⤾
 → 모듈 완료
```

### 모듈 5 · 검사 이동 및 안내  🔬  (장소: 검사실)  ※분기 가능
```
[intro] 검사실 위치·금식 확인·어지럼/불안·안전 이동
 → [branch] "의사가 검사를 지시했나요?"  → 아니오면 모듈 6으로
 → [scene] 진료실 → 검사실 이동 (엘리베이터/경사로 동선) (3D)
 → ✅[check] 검사 전후: 금식 여부 확인 / 검사 전 준비 / 검사 후 주의사항 메모 / 결과 확인 일정 기록
 → [decision] "채혈 후 어르신이 '어지럽다'고 하십니다."  ⤾
 → 모듈 완료
```

### 모듈 6 · 수납 및 약국 이용  💊  (장소: 수납창구·약국)  ※분기 가능
```
[intro] 수납·영수증/처방전·실손보험 서류·약국·복약설명 청취 지원
 → [scene] 수납창구/키오스크 (3D)
 → [kiosk] 진료비 수납: 본인확인 → 내역 확인 → 결제수단 → 영수증·처방전
 → ✅[check] 수납 확인: 진료비 내역 확인 / 본인 결제 / 영수증·처방전 수령 / 실손보험 서류 요청 여부
 → [decision] "어르신이 '네 카드로 먼저 내고 나중에 주겠다' 하십니다(금전)."  ⤾
 → [branch] "처방전이 있나요?"  → 없으면 모듈 7로
 → [scene] 약국 이동·처방전 제출 (3D)
 → [decision] "어르신이 '하루 두 번만 먹어도 되지?' 물으십니다(복약)."  ⤾
 → 모듈 완료
```

### 모듈 7 · 귀가 지원 및 기록  📝  (장소: 차량 승하차·기록지)
```
[intro] 교통수단·승하차 보조·상태 재확인·보호자 전달·기록 작성
 → [scene] 차량 승하차 (무릎 지지·허리 지지) (3D)
 → ✅[check] 귀가 업무: 교통수단 확인 / 안전 승하차 / 상태 재확인 / 보호자·기관에 종료 전달
 → [decision] "이웃이 '어디가 편찮으신 거야?' 병명을 묻습니다(개인정보)."  ⤾
 → [record] 서비스 제공 기록지 직접 작성 → 제출·보고
 → 모듈 완료
```

### 모듈 8 · 응급상황 대응 시뮬레이션  🆘  (장소 이동형)
```
[intro] 초기 대응 5원칙: ①상태확인 ②의료진/119호출 ③안전공간 ④보호자·기관 ⑤기록·보고
        장소 규칙: 병원 안=의료진·안내·보안 호출 / 병원 밖·이동 중=119 + 기관·보호자
 → [emergency 낙상]  (병원 로비)   → 초기 대응 선택 ⤾ → 신고 대상 선택(장소) ⤾
 → [emergency 의식저하·호흡곤란] (대기실) → 초기 대응 ⤾ → 신고 ⤾
 → [emergency 저혈당]  (귀가 이동 중) → 증상 인지 → 대응(응급식품/자세) ⤾ → 신고 ⤾
 → [emergency 경련]   (검사실 앞)  → 금지행위 구분(누르지 않기·입에 넣지 않기) ⤾ → 대응 ⤾
 → 공통 원칙 각인: "이용자를 혼자 두지 않는다"
 → 모듈 완료
```

## 6. 3D 공간 구성안

### 6.1 공간 목록과 학습 포인트
| 공간 | 3D 요소 | 학습 포인트 | 연계 모듈 |
|---|---|---|---|
| 병원 정문·로비 | 출입문, 안내데스크, 소파 | 만남·동선·휠체어 대여 | 1,2 |
| 무인 키오스크 | 키오스크 단말, 화면 | 접수·수납 보조, 개인정보 가림 | 2,6 |
| 원무과(유인창구) | 접수 데스크, 의자 | 유인 접수, 서류 제출 | 2,6 |
| 외래 대기실 | 대기 의자열, 전광판, TV | 대기·호출·상태 관찰 | 3 |
| 외래 진료실 | 진찰 침대, 책상, 의자 | 진료 동행, 메모, 역할 | 4 |
| 검사실 | 검사 베드, 기기 | 금식·검사 전후 관리 | 5 |
| 수납창구 | 창구, 키오스크 | 수납·영수증·처방전 | 6 |
| 약국 | 약 선반, 조제대 | 처방전 제출, 복약설명 청취 | 6 |
| 화장실·편의 | 변기, 세면대, 손잡이 | 낙상 예방, 동행 원칙 | 3 |
| 엘리베이터 | 카/문 | 휠체어 탑승(뒤로 진입) | 2,5 |
| 경사로·문턱 | 램프, 턱 | 휠체어 경사 이동, 지그재그 | 5 |
| 차량 승하차 | 차량, 좌석 | 무릎·허리 지지 승하차 | 7 |

### 6.2 3D 표현 방침 (3D-lite)
- **저폴리(low-poly) + 무료 CC0 애셋**(Kenney 가구/캐릭터) → 용량 최소·저작권 부담 없음.
- 캐릭터 2인: **동행인력(주황 조끼)**, **어르신(흰머리·지팡이)**. walk/idle 애니메이션.
- 카메라는 **자동 추적**(사용자는 드래그로 둘러보기만). 조준·이동 조작 없음.
- 텍스처 1장(컬러맵) 공유 → draw call 최소화, 모바일 60fps 목표.
- **폴백**: 애셋 로드 실패 → 절차적 박스 씬 / WebGL 미지원 → 장소 버튼 목록(3D 없이 학습 100% 가능).

### 6.3 대화형(비-3D) 처리 항목
개인정보 보호, 보호자·의료진 의사소통, 금지행위 판단 등 *공간 이해가 불필요한* 학습은 **카드형 대화·선택 UI**로 처리하여 성능·가독성을 확보한다.

## 9. 평가 및 이수 관리

### 9.1 측정 방식
- **모듈 완료**: 체크리스트 전 항목 확인 + 상황판단 응답 시 완료 처리(localStorage 저장).
- **상황 판단 채점**: 각 decision 정답 1점. **위험 행동(금지행위·응급 오답)** 선택 시 별도 `위험 플래그` 기록.
- **필수 절차 누락 체크**: 체크리스트 게이팅으로 누락 자체가 불가하나, 평가 모드에서는 "되돌아가 재확인한 횟수"를 참고 지표로 수집.

### 9.2 이수(수료) 기준 (제안)
| 항목 | 기준 |
|---|---|
| 모듈 완료 | 8개 모듈 전부 완료 |
| 상황 판단 정답률 | 전체 70% 이상 |
| 위험 행동 | 응급(모듈 8) 문항 **위험 오답 0건** 재도전으로 정정 |
| 최종 점검 퀴즈 | 8문항 중 70% 이상 |

> 기준 수치는 수행기관·지자체 지침에 맞게 `data.js`에서 조정 가능.

### 9.3 결과 화면(학습자)
- 모듈별 점수 막대, **취약 항목 목록**(틀린 수칙·위험행동), **재학습 권장 모듈** 바로가기, 이수 여부.

### 9.4 관리자용 학습현황 (검토안)
- MVP: 기기 로컬 저장(오프라인, 개인 학습). 
- 정식: 로그인 + 서버 집계 → **기관별/교육 차시별 통계**(완료율, 평균 정답률, 공통 취약 문항 Top5), CSV 내보내기. 개인정보 최소 수집(교육생 식별용 익명 ID 권장).

## 11. 개발 방식별 장단점 및 예상 비용

### 11.1 기술 스택 비교
| 방식 | 장점 | 단점 | 모바일 저사양 | 용량 | 적합도 |
|---|---|---|---|---|---|
| **Three.js (WebGL)** ★채택 | 무료·오픈소스, 웹 표준, 경량, 폴백 자유, 설치 불필요 | 대규모 게임엔 부적합 | ◎ | 소(수백KB~) | **최적** |
| Unity WebGL | 강력한 에디터·에셋 생태계 | **초기 로딩 무겁고(수십MB)** 모바일 발열·용량↑, 라이선스 | △ | 대 | 대형/네이티브 겸용 시 |
| PlayCanvas | 웹 특화, 에디터 협업, 경량 | 유료 에디터(무료 티어 제한), 종속성 | ○ | 중 | 팀 협업 3D |

### 11.2 예상 비용 (참고 범위 · 실제는 산출물 범위/기관 요구에 따라 변동)

| 3D 애셋 | CC0 무료 애셋 활용(무료) | 병원 맞춤 저폴리 제작(중) |

## 12. MVP vs 정식 개발 범위

### 12.1 MVP (현재 구현 + 본 차수 고도화 — 무료·정적 배포 가능)
- ✅ 8개 모듈 '하루 체험' 전 과정(사전준비~귀가·기록)

- ✅ 상황 판단(금지행위·안전·개인정보·자기결정 등) + 즉시 피드백
- ✅ **모듈 8 응급상황 시뮬레이션**(낙상·의식저하/호흡곤란·저혈당·경련 + 장소별 신고)
- ✅ 서비스 제공 기록지 작성 실습
- ✅ **읽어주기(TTS)**, **연습/평가 모드**, **결과·취약항목·재학습 권장**
- ✅ 진행 중간 저장(localStorage), 완료 ✓ 표시

### 12.2 정식 버전 (추가 범위)
- 로그인·수강생 관리, **서버 집계 대시보드**(기관별·차시별 통계, CSV)

- 수료증(PDF) 자동 발급, LMS 연동(SCORM/xAPI)
- 다국어(필요 시), 태블릿 키오스크 전용 모드
- 콘텐츠 관리자(CMS) 화면 — 비개발자 문구 편집 UI
- 정식 접근성 인증(웹 접근성 지침 준수 심사)

## 13. 유지보수 및 콘텐츠 수정 방식

### 13.1 콘텐츠 수정 (개발자 없이)
- **모든 교육 문구·체크리스트·문항·피드백·키오스크 흐름·기록지 양식은 `js/data.js` 단일 파일**에서 관리.
- 수정 즉시 화면 반영(재빌드 불필요, 정적 파일 교체만). 지자체·기관 지침 개정 시 해당 배열의 텍스트만 교체.
- 데이터 구조 예:
  ```js
  MODULES  // 8개 모듈 정의(제목·목표·beat 구성)
  STATIONS // 장소별 업무·주의사항
  KIOSK_FLOWS // 키오스크 단계
  SCENARIOS / EMERGENCIES // 상황판단·응급 문항 + 피드백
  RULES / QUIZ / RECORD_FORM // 수칙·퀴즈·기록지
  ```

### 13.2 배포·운영
- **정적 사이트**: 웹 서버에 폴더 업로드만으로 배포(GitHub Pages 등 무료 호스팅 가능, `.nojekyll` 포함).
- 오프라인: `vendor/`에 three.js 로컬 번들 → 최초 로드 후 자원 캐시.
- 버전 관리: Git으로 콘텐츠 이력 추적, 롤백 용이.

### 13.3 로고·브랜딩
- 시작화면 기관 로고(보건복지부/노인맞춤돌봄서비스/독거노인종합지원센터)는 저작권 문제로 **임시 SVG 심볼** 사용 중 → 배포 시 공식 로고 이미지로 교체(위치: `index.html`의 `.logo-row`).
