# lms-src — 직무교육 통계 대시보드 소스

`/lms-statistics.html`(단일 HTML 앱)을 생성하는 소스입니다. 일반 사용자는 빌드된
`lms-statistics.html` 하나만 있으면 되고, 이 폴더는 **수정·재빌드**할 때만 필요합니다.

## 구성

| 파일 | 설명 |
|------|------|
| `compute.js` | 통계 계산 엔진(순수 함수). 브라우저/Node 공용. **이수 판정·집계 로직의 핵심.** |
| `app.js` | 화면(UI) 로직 — 업로드·탭·표·필터·엑셀 내보내기·개인조회·설정 |
| `app.css` | 스타일 |
| `page.html` | HTML 골격(빌드 시 마커 치환) |
| `vendor/xlsx.full.min.js` | SheetJS (엑셀 파싱 라이브러리, 오프라인 내장용) |
| `build.js` | 위 파일들을 합쳐 `../lms-statistics.html` 단일 파일 생성 |
| `e2e.js`, `persist.js` | 실데이터 기반 브라우저 검증 스크립트(Playwright) |

## 빌드

```bash
cd lms-src
node build.js          # → ../lms-statistics.html 생성
```

`vendor/xlsx.full.min.js`를 base64로 인코딩해 HTML에 내장하므로, 결과 파일은
인터넷 없이 더블클릭만으로 동작합니다(데이터도 외부로 전송되지 않음).

## 검증(선택)

```bash
npm install            # playwright, xlsx (개발용)
node e2e.js            # 실데이터 업로드 → KPI 일치 검증
```

엔진 로직은 동일 규칙의 Python 레퍼런스와 **완전히 동일한 수치**(대상자 36,757 /
이수자 16,496 / 이수율 44.88%)를 산출함을 교차 검증했습니다.

## 이수 판정 규칙 (compute.js)

- **모수(대상자)**: 회원정보에서 `교육대상여부=Y` 이고 `상태=정상` 인 **생활지원사·전담사회복지사**
- **신규자**: 해당 직군 `필수` 과정 수료 → 이수
- **경력자**: `필수` 과정 수료 **+** `선택` 과목 수료 차시 합계 ≥ 기준(생활 13, 전담 10) → 이수
- 학생 데이터의 `수료여부=수료`를 그대로 사용, `상태=수강취소`는 제외
- 같은 (ID, 과정)이 여러 차수에 있으면 한 번이라도 수료면 수료로 인정
- 기준·차시는 앱의 **설정** 탭에서 코드 수정 없이 변경 가능
